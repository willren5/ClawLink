从异常到处理 在手机上闭环

不是只会提醒你，
而是能让你接着处理

真正有用的移动端，不是把“告警来了”推到锁屏上，而是把下一步也准备好。看到异常以后，继续查、继续问、继续做。

01

先在 Dashboard 看整体状态

先确认网关是不是在线、请求量有没有冲高、是不是只有某个 channel 或某个 agent 出问题。

02

再切到 Monitor 看日志和指标

用 rate limit、错误日志和主机曲线判断问题是网络、上游限流，还是 agent 本身卡住。

03

最后带着上下文进入 Chat

不用回到电脑，把问题直接丢给当前 agent，继续让它给出动作、文案或修复思路。

适合需要“看得住”和“接得上”的 OpenClaw Gateway 用户。不是演示面板，而是移动端操作流。
